# # 技術構成・データ設計書 (Architecture & Database Design) - V2

## 1. 技術スタック (Tech Stack)

| レイヤー | 選定技術 | 採用理由 |
| --- | --- | --- |
| **Frontend / BFF** | Next.js 15+ (App Router) | 画面とRoute Handlersの一体開発。PWA化（`@ducanh2912/next-pwa`）およびVercelでの高速配信。 |
| **Language** | TypeScript | OCR結果の複雑な文字列サジェスト・構造化ロジックを型安全に守るため。 |
| **Styling** | Tailwind CSS | チケットの「エモい」UI（半券のミシン目、グラデーション）を爆速で組むため。 |
| **Database / Auth** | Supabase | Postgresベース。将来の座席分析に向けた柔軟なクエリに対応し、RLSで個人データを保護。 |
| **OCR Engine** | Google Cloud Vision API | 劇場の暗所環境やシネコンごとのフォーマット違いを高精度に読み取り、座標情報付きでテキスト化するため。 |
| **External API** | TMDb API | OCRで抽出したタイトルから、映画情報・高画質ポスターメタデータを曖昧検索で取得。 |

## 3. データベース論理モデル (Supabase Schema)

### 3.1 `profiles` テーブル (ユーザープロファイル)

Supabase Auth の `users` テーブルと `id` (UUID) で1対1紐付け。

| カラム名 | 型 | 制約 | 説明 |
| --- | --- | --- | --- |
| `id` | `uuid` | PRIMARY KEY | Supabase Authの user_id |
| `updated_at` | `timestamp with time zone` | - | 最終更新日時 |
| `username` | `text` | - | 表示ユーザー名 |

### 3.2 `movie_tickets` テーブル (デジタル半券データ)

※技術選定に合わせて、OCR生ログ保存用のカラムや、将来の「お好み座席分析」に対応するために座席情報を最小単位に分解（パース）して保存できるカラムを追加・変更しました。

| カラム名 | 型 | 制約 | 説明 |
| --- | --- | --- | --- |
| `id` | `uuid` | PRIMARY KEY, DEFAULT: gen_random_uuid() | チケット一意ID |
| `user_id` | `uuid` | REFERENCES profiles(id), NOT NULL | 所有ユーザーID |
| `theater_name` | `text` | NOT NULL | ユーザー確認済みの劇場名 (例: 'TOHOシネマズ新宿') |
| `screen_name` | `text` | NULLABLE | スクリーン名・番号 (例: 'SCREEN 7', 'IMAX') 【座席分析用】 |
| `show_timestamp` | `timestamp with time zone` | NOT NULL | 上映日時 (OCRまたは手動補完) |
| `movie_title` | `text` | NOT NULL | TMDbから確定させた正式な映画タイトル |
| `poster_path` | `text` | - | TMDbのポスター画像フルURLまたはパス |
| `seat_raw` | `text` | NULLABLE | 座席番号の文字列表現全般 (例: 'J-12') |
| `seat_row` | `text` | NULLABLE | 分解した座席の「列/行」 (例: 'J') 【座席分析用】 |
| `seat_number` | `integer` | NULLABLE | 分解した座席の「番/列」 (例: 12) 【座席分析用】 |
| `rating` | `numeric` | DEFAULT: 0 | 5段階の星評価 (0.5刻み) |
| `memo` | `text` | - | ユーザーのひと言メモ |
| `raw_ocr_text` | `text` | NULLABLE | Cloud Vision APIから返ってきた生のテキストデータ (解析・調整ログ用) |
| `created_at` | `timestamp with time zone` | DEFAULT: now() | 登録日時 |

## 4. セキュリティ方針 (Row Level Security)

SupabaseのRLSを有効化し、以下のポリシーを適用する。

* **Policy:** `auth.uid() == user_id`
* **対象操作:** `SELECT`, `INSERT`, `UPDATE`, `DELETE`
* **効果:** 自分以外のユーザーが登録したチケットデータやOCR履歴には、API経由であっても絶対にアクセスできない状態を担保する。

### 今回の変更ポイント

1. **インフラ・外部APIに「Cloud Vision API」を追加：** トリガーがQRからOCRに変わったため、アーキテクチャおよびフローに組み込みました。また、これに伴い旧スキーマの `brand` や `theater_code` などのQR依存カラムを廃止または汎用的な形に変更しました。
2. **座席分析用カラムの拡張：** チームのフィードバックから決定した「座席傾向分析」が将来スムーズに行えるよう、`seat_row`（列: J）と `seat_number`（番: 12）に切り分けて保存できる構造に設計を変更しています。
3. **生ログの変更：** `raw_qr_string` だったカラムを、OCRのチューニングやサジェスト精度の改善に再利用できるよう `raw_ocr_text` に変更しました。
