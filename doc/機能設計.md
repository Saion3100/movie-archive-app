# 機能設計・ロジック仕様書 (Functional Design)

## 1. コアロジック：QRコードパース仕様
メイン関数 `parseQRCode(raw: string)` により、入力された文字列を検知して以下の3ドメインにルーティングする。

### 1.1 京成ローザ (KEISEI)
- **トリガー:** 文字数が完全に「25桁」かつ「すべて数字」。
- **切り出しルール:**
  - `1-4桁`: 劇場コード (例: `8001`)
  - `5-6桁`: 上映西暦下2桁 (`26` -> 2026年として処理)
  - `7-10桁`: 月日MMDD (`0530` -> 5月30日)
  - `11-25桁`: スケジュールIDおよびシリアル番号を結合し、`reserveId` とする。

### 1.2 T・ジョイ (TJOY)
- **トリガー:** 文字数が完全に「15桁」かつ「すべて数字」。
- **切り出しルール:**
  - `1-4桁`: 劇場コード (例: `5362`)
  - `5-15桁`: 予約ID/チケット一意ID (※日付・時間情報を持たないため、showTimestampは `null` 確定)

### 1.3 USシネマ (USCINEMA)
- **トリガー:** 先頭がシステム共通コード `0002` で始まり、文字列全体の長さが `29` 以上。
- **切り出しルール:**
  - `5-8桁`: 劇場コード (例: `0008`)
  - `9-14桁`: 予約番号
  - `15-26桁`: 日時情報 YYYYMMDDHHMM から Date オブジェクトを完全生成。
  - `30桁目以降`: 空白区切りの可変テキスト。配列の0番目を `movieTitleShort` として抽出し、TMDbの自動検索のシード値とする。

## 2. 画面フロー・コンポーネント設計 (Dashboard Flow)
[ トップ画面 (Home) ]
├── ① チケットスキャンフォーム (Scan Form)
│     └── 入力値を utils/ticketParser で即座にバリデーション
├── ② 作品メタデータ紐付けセクション (Movie Matcher)
│     ├── 自動検索 (USシネマの場合、自動で /api/search をコール)
│     └── 手動検索 (ローザ/T・ジョイの場合、検索窓から自由に入力)
└── ③ リアルタイムチケットプレビュー (Ticket Preview)
└── 選択されたポスター画像と、パースされた劇場データをブレンドして描画


### 2.1 1次バリデーション仕様 (`validateTicket`)
- **劇場コード合致チェック:** 選択された入場ゲート（システム）の劇場コードと、チケット内のコードが一致するか。
- **上映日チェック:** チケット内の日付が「本日（サーバー/クライアントの現在日）」と一致するか。
  - *例外:* T・ジョイ系列は日付を持たないため、劇場コードのみをチェックし、結果オブジェクトの `requiresDbCheck` フラグを `true` で返す
