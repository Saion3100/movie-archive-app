# 要求分析書 (Requirements Analysis)

## 1. プロダクト概要
映画館で発券される「紙の入場券（半券）」は、時間が経つと感熱紙のインクが薄れて消えてしまう。本プロダクトは、映画館を出た直後の熱量のまま、チケットのQRコードや文字列をスキャンすることで、一生色褪せない「デジタル半券（資産）」として美しくアーカイブ・コレクションできる個人向けWebアプリケーションである。

### 1.1 ペルソナ（想定ユーザー）
- **佐藤さん (映画コアファン)**
  - 月に数回〜十数回映画館に足を運ぶ。
  - 鑑賞の証として紙の半券を保管しているが、整理が面倒で紛失しがち。
  - 映画館を出た直後のロビーで、余韻に浸りながら「爆速で」記録を完了させたい。
  - 自分の集めた「チケット棚」を眺めてニヤニヤしたい、たまにSNSにスクショを上げて自慢したい。

## 2. コア体験と要求事項 (User Stories)

### 2.1 ユーザー要求 (User Requirements)
- **【RQ-01】手入力の極小化:** 映画館ロビーを歩きながらでも、5秒以内に登録が完了すること。チケットの文字をポチポチ入力させない。
- **【RQ-02】情緒的価値（エモさ）の最大化:** 登録されたデータは単なるリストではなく、本物の映画ポスターと上映データが美しく融合した「デジタル半券」として生成されること。
- **【RQ-03】手元データの即時互換:** 千葉エリアを中心とした主要シネコン（京成ローザ、T・ジョイ、USシネマ等）のチケットが、文字数のルールだけで即座に判別・パースされること。

### 2.2 システム要求 (System Requirements)
- **QR/シリアル解析機能:** 入力された数字の羅列から「劇場」「上映日」「予約ID」を即座に分解する。
- **外部映画データ連携:** 分解されたわずかな情報（あるいは手動検索キー）から、TMDb APIを用いて正式な「作品タイトル」「ポスター画像URL」を取得する。
- **永続化と認証:** ユーザーごとにコレクションを安全に分離して保存できること（将来的なマルチユーザー化を見据える）。

## 3. スコープ（MVPで「やること」「やらないこと」）

### 3.1 やること（MVPスコープ）
1. QRコードの数値/文字列の手動ペーストによる即時解析。
2. TMDb APIと連動した、作品ポスターの自動検索およびマニュアル選択紐付け。
3. 縦型デジタル半券のリアルタイムプレビューUI。
4. Supabase DBへのチケット情報の保存および一覧表示。

### 3.2 やらないこと（将来フェーズ）
- スマホカメラによる直接的な「リアルタイムQRコードビデオ・スキャン」（初期は画像読み込み、またはテキスト入力で代替）。
- チケット券面のOCR（文字認識）による自動解析の高度化（初期はUSシネマのテキストパターンのみ対応）。
- ユーザー同士のSNS機能、コメント・タイムライン機能。
